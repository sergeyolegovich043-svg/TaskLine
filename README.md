# TaskLine / Service Desk MVP

Статический MVP enterprise-системы TaskLine для управления проектами и заявками. Система объединяет Service Desk, проекты, спринты, backlog, бюджетирование, Gantt, трудозатраты и многоуровневые согласования. Центральная сущность — задача: она одновременно является заявкой Service Desk, задачей разработки, элементом спринта, бюджетной строкой, workflow-объектом и объектом согласования.

## Запуск

```powershell
python -m http.server 4173 --bind 127.0.0.1
```

Откройте [http://127.0.0.1:4173/](http://127.0.0.1:4173/). Данные хранятся в `localStorage`. Кнопка «Сбросить демо» пересоздает проекты, спринты, задачи и бюджеты.

## Основные модули

- Главный экран после входа: карточки проектов «Мобильное приложение» и «Интернет портал».
- Карточка проекта: задачи, задачи в работе, задачи на согласовании, бюджет план/факт, процент выполнения.
- Страница проекта: вкладки «Бюджет», «Заявки», «Ведение проекта».
- Бюджет: годовой бюджет, полугодия, месяцы план/факт, задачи со стоимостью и часами, диаграмма Ганта.
- Бюджетная модель: стоимость по умолчанию считается как `часы × ставка`, при необходимости допускается ручная корректировка.
- Заявки: список, фильтры, поиск, карточка задачи, файлы, чат, история, согласования.
- Ведение проекта: вертикальный Jira Backlog со спринтами сверху вниз и Backlog внизу.
- Drag-and-drop: перенос задач внутри спринтов, между спринтами и в Backlog.
- Workflow задачи: «Создано» → «Анализ» → «Оценка (Подрядчик)» → «Согласование оценки (Техдир)» → «Согласование бюджета (РП)» → «Согласование старта (Топ менеджер)» → «В работе (Разработка)» → «Тестирование (Заказчик)» → «Готов к релизу» → «Демонстрация» → «Закрыто».
- Многоуровневое согласование: отдельные approval-записи по этапам, ролям, пользователям, статусам, комментариям и датам.
- Связность данных: изменения задачи сразу отражаются во вкладках бюджета, заявок и ведения проекта.

## Проверяемый сценарий

1. Войти как Подрядчик.
2. Открыть проект «Интернет портал».
3. Во вкладке «Ведение проекта» перенести задачу из Backlog в спринт.
4. Изменить часы или стоимость задачи.
5. Проверить, что бюджет и Gantt во вкладке «Бюджет» обновились.
6. Открыть карточку задачи, отправить этап на согласование нужной роли.
7. Войти как Технический директор, Budget Owner, Топ-менеджер или Заказчик в зависимости от этапа и согласовать задачу.
8. Войти как Подрядчик или Менеджер проекта и перевести задачу на следующий статус.
